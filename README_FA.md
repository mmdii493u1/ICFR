# ICFR (نمایش درخواست‌های باقی‌مانده Cloudflare)

با استفاده از ICFR می‌توانید مقدار درخواست‌های باقی‌مانده (Remaining Requests) اکانت یا اکانت‌های Cloudflare خود را مستقیماً از طریق تلگرام مشاهده کنید.

ICFR یک ربات متن‌باز تلگرام است که به شما اجازه می‌دهد یک یا چند اکانت Cloudflare را ثبت کرده و تنها با چند کلیک، میزان درخواست‌های باقی‌مانده هر اکانت را بررسی کنید.

---

# پیش‌نیازها

قبل از استفاده از این پروژه، موارد زیر را آماده کنید:

* یک حساب Cloudflare
* توکن ربات تلگرام که از **@BotFather** دریافت می‌کنید.
* آیدی عددی تلگرام که از **@userinfobot** دریافت می‌کنید.

---

# راه‌اندازی Cloudflare Worker

## مرحله ۱ - ایجاد Worker

وارد پنل Cloudflare شوید.

از منوی سمت چپ مسیر زیر را باز کنید:

```text
Compute → Workers & Pages
```

در گوشه بالا سمت راست روی گزینه **Create Application** کلیک کنید.

سپس گزینه زیر را انتخاب کنید:

```text
Start with Hello World
```

پس از ساخته شدن Worker، روی **Edit Code** کلیک کنید.

> **نکته**
>
> اگر گزینه **Edit Code** را مشاهده نمی‌کنید، ابتدا Worker ساخته‌شده خود را از لیست انتخاب کنید.

---

## مرحله ۲ - جایگزین کردن کد Worker

کد پیش‌فرض فایل `worker.js` را به‌طور کامل حذف کنید.

سپس یکی از دو روش زیر را انجام دهید:

* فایل `worker.js` موجود در این پروژه را آپلود کنید.
* یا محتوای فایل `worker.js` پروژه را کپی کرده و به جای کد پیش‌فرض قرار دهید.

---

## مرحله ۳ - تنظیم شناسه مدیر ربات

قبل از Deploy کردن Worker، متغیر زیر را که در ابتدای فایل `worker.js` قرار دارد ویرایش کنید:

```javascript
CONFIG_ADMIN_TELEGRAM_ID
```

مقدار این متغیر را با **آیدی عددی تلگرام** حسابی که قرار است مدیر ربات باشد جایگزین کنید.

تنها این حساب تلگرام به پنل مدیریت ربات دسترسی خواهد داشت.

برای دریافت آیدی عددی تلگرام خود می‌توانید به ربات زیر پیام دهید:

```text
@userinfobot
```

---

## مرحله ۴ - Deploy کردن Worker

پس از جایگزین کردن کدها و تنظیم شناسه مدیر، روی گزینه زیر کلیک کنید:

```text
Deploy
```

چند لحظه صبر کنید تا Cloudflare عملیات Deploy را به پایان برساند.

---

# ایجاد Secretهای Worker

پس از پایان Deploy، وارد Worker شوید و مسیر زیر را باز کنید:

```text
Settings → Variables and Secrets
```

روی **Add** کلیک کرده و Secretهای زیر را ایجاد کنید.

---

## Secret اول

**نام متغیر**

```text
TELEGRAM_BOT_TOKEN
```

**مقدار**

ابتدا توسط ربات **@BotFather** یک ربات تلگرام ایجاد کنید.

دستور زیر را اجرا کنید:

```text
/newbot
```

مراحل ساخت ربات را تکمیل کنید.

پس از پایان، BotFather یک **HTTP API Token** در اختیار شما قرار می‌دهد.

نمونه:

```text
123456789:AAxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

این مقدار را به عنوان مقدار متغیر **TELEGRAM_BOT_TOKEN** وارد کنید.

سپس روی **Deploy** کلیک کنید تا تغییرات ذخیره شوند.

---

## Secret دوم

دوباره روی **Add** کلیک کرده و یک Secret جدید ایجاد کنید.

**نام متغیر**

```text
BOT_PASSWORD
```

**مقدار**

یک رمز عبور دلخواه و امن انتخاب کنید.

این رمز هنگام ورود اولیه به ربات از شما درخواست خواهد شد.

> ⚠️ این رمز را در محل امنی نگهداری کنید؛ زیرا برای ورود به ربات به آن نیاز خواهید داشت.

پس از ذخیره این متغیر نیز مجدداً روی **Deploy** کلیک کنید.

# ایجاد KV Namespace

این ربات اطلاعات خود را در **Cloudflare Workers KV** ذخیره می‌کند.

وارد پنل Cloudflare شوید و مسیر زیر را باز کنید:

```text
Storage & Databases → Workers KV
```

روی **Create Instance** کلیک کنید.

یک نام دلخواه برای KV Namespace خود وارد کرده و روی **Create** کلیک کنید.

---

# اتصال KV Namespace به Worker

پس از ایجاد KV Namespace، دوباره وارد بخش زیر شوید:

```text
Compute → Workers & Pages
```

Worker ساخته‌شده خود را باز کنید.

سپس مسیر زیر را دنبال کنید:

```text
Settings → Bindings
```

روی **Add Binding** کلیک کنید.

تنظیمات را به شکل زیر انجام دهید.

### نوع Binding

```text
KV Namespace
```

### نام متغیر

```text
KV
```

> **نکته مهم**
>
> مقدار **Variable Name** باید دقیقاً برابر با `KV` و با حروف بزرگ (Uppercase) باشد.

### KV Namespace

KV Namespace که در مرحله قبل ایجاد کرده‌اید را انتخاب کنید.

در پایان روی **Add Binding** کلیک کنید.

# تنظیم Webhook تلگرام

اکنون باید به تلگرام معرفی کنید که تمامی پیام‌های دریافتی ربات را به Cloudflare Worker شما ارسال کند.

مرورگر خود را باز کرده و آدرس زیر را وارد کنید:

```text
https://api.telegram.org/bot<YOUR_BOT_TOKEN>/setWebhook?url=<YOUR_WORKER_URL>
```

نمونه:

```text
https://api.telegram.org/bot123456789:ABCdefGhIJKlmNoPQRsTUVwxyZ/setWebhook?url=https://my-bot.my-subdomain.workers.dev
```

مقادیر زیر را جایگزین کنید:

* `<YOUR_BOT_TOKEN>` را با توکنی که از **@BotFather** دریافت کرده‌اید جایگزین کنید.
* `<YOUR_WORKER_URL>` را با آدرس Cloudflare Worker خود جایگزین کنید.

برای مشاهده آدرس Worker کافی است وارد Worker شوید و روی گزینه **Visit** کلیک کنید.

اگر همه مراحل را به‌درستی انجام داده باشید، پاسخ زیر نمایش داده خواهد شد:

```json
{
  "ok": true,
  "result": true,
  "description": "Webhook was set"
}
```

تبریک! 🎉

اکنون ربات تلگرام شما با موفقیت به Cloudflare Worker متصل شده است و از این پس تمام پیام‌های دریافتی ربات مستقیماً به Worker ارسال خواهند شد.

# افزودن اکانت Cloudflare

تنظیمات Cloudflare اکنون به پایان رسیده است.

ربات تلگرام خود را باز کرده و دستور **/start** را اجرا کنید.

در اولین ورود، رمز عبوری که در متغیر **BOT_PASSWORD** تعیین کرده‌اید از شما درخواست می‌شود.

پس از وارد کردن رمز، گزینه زیر را انتخاب کنید:

```text
Add Cloudflare Account
```

ربات اطلاعات زیر را از شما دریافت خواهد کرد.

---

## مرحله ۱ - نام اکانت

ابتدا یک نام دلخواه برای اکانت Cloudflare خود وارد کنید.

این نام فقط داخل ربات نمایش داده می‌شود و به شما کمک می‌کند در صورت ثبت چند اکانت، آن‌ها را راحت‌تر تشخیص دهید.

برای مثال:

```text
Personal Account
Production
Main Account
```

## مرحله ۲ - Account ID

وارد پنل Cloudflare شوید و مسیر زیر را باز کنید:

```text
Compute → Workers & Pages
```

شناسه **Account ID** در قسمت پایین سمت راست صفحه نمایش داده می‌شود.

آن را کپی کرده و برای ربات ارسال کنید.

---

## مرحله ۳ - ایجاد API Token

در پنل Cloudflare روی آیکون پروفایل خود در گوشه بالا سمت راست کلیک کنید.

سپس مسیر زیر را دنبال کنید:

```text
Profile → API Tokens
```

روی گزینه زیر کلیک کنید:

```text
Create Token
```

سپس گزینه زیر را انتخاب کنید:

```text
Create Custom Token
```

> **نکته مهم**
>
> قبل از ادامه، حتماً یک نام برای توکن وارد کنید؛ در غیر این صورت امکان رفتن به مرحله بعد وجود نخواهد داشت.

سطح دسترسی (Permission) را به شکل زیر تنظیم کنید:

| Resource | Permission        | Access |
| -------- | ----------------- | ------ |
| Account  | Account Analytics | Read   |

سپس روی گزینه زیر کلیک کنید:

```text
Continue to Summary
```

و در نهایت:

```text
Create Token
```

توکن ایجادشده در بالای صفحه نمایش داده خواهد شد.

آن را کپی کرده و برای ربات ارسال کنید.

---

# مشاهده درخواست‌های باقی‌مانده

پس از اینکه اطلاعات اکانت با موفقیت ثبت شد، از منوی ربات گزینه زیر را انتخاب کنید:

```text
Remaining Requests
```

ربات میزان درخواست‌های باقی‌مانده (Remaining Requests) اکانت Cloudflare انتخاب‌شده را نمایش خواهد داد.

اگر چند اکانت ثبت کرده باشید، هر اکانت با همان نامی که هنگام افزودن انتخاب کرده‌اید نمایش داده می‌شود تا بتوانید به‌راحتی درخواست‌های باقی‌مانده هر اکانت را مدیریت و بررسی کنید.
